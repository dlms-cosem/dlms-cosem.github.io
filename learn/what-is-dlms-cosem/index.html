What is DLMS/COSEM?

DLMS/COSEM is the international standard (IEC 62056) for smart metering data exchange, maintained by the DLMS User Association. It's used globally for electricity, gas, water, and heat metering applications.

DLMS vs COSEM: Understanding the Difference

Many people use "DLMS/COSEM" as a single term, but they actually serve different purposes in the communication stack:

DLMS (Device Language Message Specification)

  • Layer: Data Link Layer (in OSI model)
  • Function: Defines HOW data is exchanged
  • Scope: Transport protocols and communication methods
  • Documentation: Specified in the "Green Book"

COSEM (Companion Specification for Energy Metering)

  • Layer: Presentation Layer (in OSI model)
  • Function: Defines WHAT data exists
  • Scope: Data model, object structure, and data organization
  • Documentation: Specified in the "Blue Book"

How They Work Together

Think of it this way:

  • COSEM = The language and vocabulary (what you say)
  • DLMS = The delivery method (how you say it)

Together, they form a complete object-oriented framework for meter communication.

Core Concepts

1. The COSEM Object Model

Everything in a DLMS/COSEM meter is represented as an object:

  • Each object belongs to an interface class (like a template)
  • Objects have attributes (data values)
  • Objects have methods (operations you can perform)

For example, a "Register" object might have:

  • Attributes: current value, unit, scaler
  • Methods: reset, capture

2. OBIS Identification System

Every piece of data in a meter has a unique address called an OBIS code.

OBIS codes use this format: A.B.C.D.E.F

  • A = Medium (1=Electricity, 7=Gas, 8=Water)
  • B = Channel number
  • C = Measurement type
  • D = Processing type (7=Instantaneous, 8=Maximum)
  • E = Tariff (0=Total, 1-4=Tariff number)
  • F = Billing period (255=Current)

Example: 1.0.1.7.0.255 = Electricity, Total Active Power, Instantaneous, Current billing period

OBIS Code Decoder

3. Communication Profiles

DLMS/COSEM works over multiple transport layers:

  • Serial (HDLC): Traditional wired connection
  • TCP/IP: Network-based communication
  • Wireless: GPRS, LTE, LoRaWAN
  • PLC: Power Line Communication
  • M-Bus: European metering bus

Security Features

Modern smart meters must be secure. DLMS/COSEM includes:

  • Authentication: Verify device identity
  • Encryption: AES-GCM encryption
  • Access Control: Role-based permissions
  • Digital Signatures: Ensure data integrity
  • Key Management: Secure key distribution and rotation

Standards & Governance

DLMS User Association

The DLMS UA maintains the specification:

  • 200+ members worldwide
  • 45+ countries represented
  • 1,700+ certified products

IEC 62056 Standards

The official international standard suite includes:

  • Blue Book: COSEM object model
  • Green Book: DLMS communication
  • Companion Specs: Industry-specific profiles
