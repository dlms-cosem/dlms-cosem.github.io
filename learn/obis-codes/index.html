Understanding OBIS Codes

OBIS (Object Identification System) is the naming system used in COSEM to uniquely identify every piece of data in a meter. Think of it as the "address" for meter data.

ℹ️

Every OBIS code tells you exactly what data you're looking at, where it comes from, and how it's processed.

OBIS Code Structure

OBIS codes have 6 groups labeled A through F, separated by dots:

A . B . C . D . E . F

Each group has a specific meaning:

Group A: Medium

Identifies the type of utility or measurement:

ValueMedium
0Abstract objects (non-physical)
1Electricity
4Heat cost allocator
5Cooling
6Heat
7Gas
8Water (cold)
9Water (hot)

Group B: Channel

Physical or logical channel number:

  • 0-255: Channel number (0 = no channel, 1-255 = specific channels)
  • Often 0 for single-channel meters
  • Multi-phase meters use different channels for each phase

Group C: Physical Quantity

What is being measured:

Electricity Examples:

ValueQuantity
1Active power/energy (kW/kWh)
2Reactive power/energy (kVAR/kVARh)
3Apparent power/energy (kVA/kVAh)
4Current (A)
11-73Phase voltages and currents
13Power factor
14Frequency (Hz)
81Angles
91Neutral current

Gas/Water Examples:

ValueQuantity
0-9Volume, mass, density, etc.

Group D: Processing Type

How the value is calculated or processed:

ValueType
0Billing value
1-5Cumulative values
6Value at measurement period end
7Instantaneous value
8Time integral (average)
9Maximum value
10Minimum value
27-29Profile values

Group E: Tariff/Rate

Which tariff or rate applies:

ValueTariff
0Total (all tariffs)
1Tariff 1
2Tariff 2
3Tariff 3
4Tariff 4

Group F: Billing Period

Historical billing period:

ValuePeriod
0-99Historical period (0 = last, 99 = 99 periods ago)
255Current billing period

Common OBIS Codes

Electricity Metering

Energy Registers:

  • 1.0.1.8.0.255 - Total active energy import (kWh)
  • 1.0.1.8.1.255 - Active energy import, Tariff 1
  • 1.0.2.8.0.255 - Total active energy export (kWh)
  • 1.0.3.8.0.255 - Total reactive energy import (kVARh)
  • 1.0.4.8.0.255 - Total reactive energy export (kVARh)

Instantaneous Values:

  • 1.0.1.7.0.255 - Active power import (kW)
  • 1.0.2.7.0.255 - Active power export (kW)
  • 1.0.31.7.0.255 - Current phase L1 (A)
  • 1.0.32.7.0.255 - Voltage phase L1 (V)
  • 1.0.14.7.0.255 - Frequency (Hz)
  • 1.0.13.7.0.255 - Power factor

Maximum Demand:

  • 1.0.1.6.0.255 - Maximum demand active power import (kW)
  • 1.0.1.6.0.0 - Max demand from last billing period

Three-Phase Values:

  • 1.0.31.7.0.255 - Current L1
  • 1.0.51.7.0.255 - Current L2
  • 1.0.71.7.0.255 - Current L3
  • 1.0.32.7.0.255 - Voltage L1
  • 1.0.52.7.0.255 - Voltage L2
  • 1.0.72.7.0.255 - Voltage L3

Gas Metering

  • 7.0.3.0.0.255 - Volume (total)
  • 7.0.3.7.0.255 - Instantaneous volume flow
  • 7.0.11.0.0.255 - Temperature
  • 7.0.42.0.0.255 - Pressure

Water Metering

  • 8.0.1.0.0.255 - Water volume (total)
  • 8.0.1.7.0.255 - Instantaneous flow rate
  • 8.0.3.0.0.255 - Forward volume
  • 8.0.4.0.0.255 - Reverse volume (leakage)

Abstract Objects (A=0)

These are system/configuration objects:

  • 0.0.1.0.0.255 - Clock
  • 0.0.40.0.0.255 - Association object (object list)
  • 0.0.42.0.0.255 - SAP assignment
  • 0.0.96.1.0.255 - Meter serial number
  • 0.0.96.1.1.255 - Manufacturer ID
  • 0.0.96.1.2.255 - Meter type
  • 0.0.96.3.10.255 - Disconnect control
  • 0.0.97.97.0.255 - Error register
  • 0.0.99.1.0.255 - Load profile

Reading OBIS Codes

Let's practice reading OBIS codes:

Example 1: 1.0.1.8.0.255

  • A = 1: Electricity
  • B = 0: No specific channel
  • C = 1: Active energy
  • D = 8: Time integral (cumulative)
  • E = 0: Total (all tariffs)
  • F = 255: Current billing period

Meaning: Total cumulative active energy imported (electricity consumption in kWh)

Example 2: 1.0.32.7.0.255

  • A = 1: Electricity
  • B = 0: No specific channel
  • C = 32: Voltage L1
  • D = 7: Instantaneous value
  • E = 0: Total/not applicable
  • F = 255: Current

Meaning: Current voltage on phase L1 (V)

Example 3: 7.0.3.7.0.255

  • A = 7: Gas
  • B = 0: No specific channel
  • C = 3: Volume flow
  • D = 7: Instantaneous value
  • E = 0: Total
  • F = 255: Current

Meaning: Current gas flow rate (m³/h)

Example 4: 1.0.1.8.1.0

  • A = 1: Electricity
  • B = 0: No specific channel
  • C = 1: Active energy
  • D = 8: Cumulative
  • E = 1: Tariff 1
  • F = 0: Last billing period

Meaning: Active energy import for Tariff 1 from the last completed billing period

OBIS in Practice

Building a Dashboard

To create a real-time dashboard, you'd read:

const dashboardObjects = [
  '1.0.1.8.0.255',  // Total energy
  '1.0.1.7.0.255',  // Current power
  '1.0.32.7.0.255', // Voltage L1
  '1.0.52.7.0.255', // Voltage L2
  '1.0.72.7.0.255', // Voltage L3
  '1.0.31.7.0.255', // Current L1
  '1.0.51.7.0.255', // Current L2
  '1.0.71.7.0.255', // Current L3
  '1.0.14.7.0.255', // Frequency
  '1.0.13.7.0.255', // Power factor
];

Reading Tariff Data

For multi-tariff meters:

const tariffObjects = [
  '1.0.1.8.0.255',  // Total (all tariffs)
  '1.0.1.8.1.255',  // Tariff 1 (peak)
  '1.0.1.8.2.255',  // Tariff 2 (off-peak)
  '1.0.1.8.3.255',  // Tariff 3 (shoulder)
];

Historical Billing Data

To get last 12 months of billing data:

const billingHistory = [];
for (let period = 0; period < 12; period++) {
  billingHistory.push(`1.0.1.8.0.${period}`);
}
// Results: 1.0.1.8.0.0, 1.0.1.8.0.1, ..., 1.0.1.8.0.11

OBIS Extensions

Some manufacturers extend OBIS codes for custom features:

  • Ranges 128-254 in groups C, D, or E are manufacturer-specific
  • Always check manufacturer documentation
  • Standard codes should be preferred when available
⚠️

Manufacturer-specific OBIS codes may not work across different meter brands!

Finding OBIS Codes

Method 1: Association Object

Read object 0.0.40.0.0.255 (Association):

  • Returns complete list of accessible objects
  • Shows class IDs and OBIS codes

Method 2: Documentation

Check:

  • Meter manufacturer's datasheet
  • DLMS Blue Book (IEC 62056-6-2)
  • OBIS code lists online

Method 3: DLMS Test Tools

Use tools like Gurux Director:

  • Connect to meter
  • Browse object tree
  • View all available OBIS codes

Common Patterns

Pattern 1: Phases L1, L2, L3

Many values repeat for each phase:

1.0.32.7.0.255  -> Voltage L1
1.0.52.7.0.255  -> Voltage L2
1.0.72.7.0.255  -> Voltage L3

Pattern: 1.0.[32,52,72].7.0.255

Pattern 2: Import/Export

Energy flow direction:

1.0.1.8.0.255  -> Active energy import
1.0.2.8.0.255  -> Active energy export

Pattern: 1.0.[1=import, 2=export].8.0.255

Pattern 3: Active/Reactive/Apparent

Power types:

1.0.1.7.0.255  -> Active power (W)
1.0.3.7.0.255  -> Reactive power (VAR)
1.0.9.7.0.255  -> Apparent power (VA)

Pattern: 1.0.[1,3,9].7.0.255

Tips for Working with OBIS

1. Use Standard Codes

Stick to defined codes for maximum compatibility.

2. Document Custom Codes

If using manufacturer extensions, document them clearly.

3. Group Related Reads

Read related OBIS codes together (e.g., all phase voltages).

4. Cache Code Meanings

Build a lookup table to avoid repeated parsing.

5. Validate Codes

Check format before sending requests to avoid errors.

OBIS Code Validation

A valid OBIS code must:

  • Have exactly 6 groups
  • Each group is 0-255
  • Groups separated by dots
  • Follow standardized meaning
function validateObis(code) {
  const parts = code.split('.');
  if (parts.length !== 6) return false;

  for (const part of parts) {
    const num = parseInt(part, 10);
    if (isNaN(num) || num < 0 || num > 255) return false;
  }

  return true;
}

Summary

OBIS codes provide:

  • Unique identification for every meter value
  • Standardized naming across vendors
  • Self-documenting structure
  • Flexibility for extensions
  • Hierarchy for organization

Mastering OBIS codes is essential for efficient DLMS/COSEM development!
